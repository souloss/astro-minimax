## 关于本文

这篇文章是一份完整的个人技术博客系统架构参考文档，涵盖了构建现代化技术博客所需了解的方方面面。

文档以三层信息架构组织内容：

- **概览层** — 快速了解构建博客需要关注的 10 个核心领域
- **详情层** — 每个领域的关键概念、特性和分类
- **解释层** — 对每个知识点的全方位讲解，包含实践指南和工具推荐

### 架构概览

| 层次 | 名称 | 核心关注 |
|------|------|----------|
| L1 | 内容模型层 | 内容类型、分类体系、Frontmatter 规范 |
| L2 | 创作与生产层 | 工具链、AI 辅助写作、质量保证 |
| L3 | 呈现与体验层 | 性能、阅读增强、交互组件、无障碍 |
| L4 | 发现与分发层 | SEO、结构化数据、RSS、Newsletter |
| L5 | 互动与社区层 | 评论系统、互动形态、知识回流 |
| L6 | AI 能力层 | 构建时 AI、运行时 AI、使用边界 |
| L7 | 信任与品牌层 | 作者身份、内容透明度、隐私治理 |
| L8 | 工程与部署层 | 框架选型、CI/CD、安全配置 |
| L9 | 监控与运营层 | 搜索监控、流量分析、自动化运维 |
| L10 | 可持续发展层 | 成本控制、内容节奏、价值变现 |
