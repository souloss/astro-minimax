思维导图是组织思路、规划项目和可视化概念关系的强大工具。借助 Markmap，你可以直接在 Markdown 文件中使用 ` ```markmap ` 代码块内的简单缩进列表来创建交互式、可折叠的思维导图。

---

## Markmap 工作原理

Markmap 将 Markdown 列表转换为交互式思维导图。语法非常直观：

- 使用标准 Markdown 标题或列表语法
- 缩进定义父子关系
- 每一行成为导图中的一个节点
- 节点可折叠，支持交互
- 节点内可使用 **粗体**、*斜体*、`代码` 和链接

只需将内容包裹在 ` ```markmap ` 代码块中，即可渲染为精美的交互式 SVG 思维导图。

## 高效思维导图技巧

以下是创建清晰实用思维导图的一些指南：

1. **节点保持简洁** — 每个节点应是一个短语，而非整段话。
2. **控制层级深度** — 3-4 层通常是最佳的。更深的嵌套会变得难以阅读。
3. **平衡各分支** — 尽量让子节点的分布大致均匀。
4. **善用格式** — 粗体表示强调，`代码` 表示技术术语，链接用于引用。
5. **归组相关概念** — 同一层级的节点应当相互关联，结构上相互平行。
6. **先宏观后细节** — 中心是最笼统的概念，具体细节分布在外围。

Markmap 非常适合规划、头脑风暴和文档编写。结合 [Mermaid 图表](/zh/blog/_examples/mermaid-diagrams) 等其他可视化工具使用，可以创建流程图和时序图。
